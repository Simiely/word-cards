# 开发日志

## 2026-06-17 核心功能

**滑动窗口预加载**：首屏预加载 4 张（当前 + 后 3），图片优先并行 → 音频 `fetch()` 不阻塞 → 200ms 防抖 + `navLock` 防冲突。

| Bug | 根因 | 解决 |
|-----|------|------|
| 加载动画永不消失 | Promise 卡死，超时兜底有条件漏洞 | 无条件超时 + `done` 防重入 + 6s 总兜底 |
| `var` 变量提升 | for 循环共享引用 | IIFE 闭包隔离 |
| `preloadBatch` 闭包 | `.then()` 回调索引是循环结束值 | IIFE 捕获当前 `idx` |
| 翻页闪白 | 未预加载图片瞬间空白 | `opacity:0` → `onload` 淡入 |
| loading 遮罩叠加 | 快速翻页多 loading 并存 | `navLock` 互斥锁 |

**架构**：分组预加载（废弃）→ 无预加载（过渡）→ 滑动窗口（最终）

### 经验教训

1. 超时兜底不能有条件；`var` 在 for 里用 IIFE 隔离
2. 硬编码从数据源动态读取；SW Cache 持久化优于内存缓存
3. 公共模块先统一接口（`speak(lang, animal)`），再重构
4. Canvas 采样要缩小（200px 足够）；fetch() 比 new Audio() 更轻
5. 手势冲突按 DOM 区域隔离；重构先于新功能
